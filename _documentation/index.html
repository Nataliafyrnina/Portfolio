2. Installation

Installing CIRCULAR is a very simple and straight forward job. Please consider the following:

2.1. Requirements

PHP 5 Support

In order for the contact form of the template to function properly, your server needs to be setup to interpret PHP scripts, but that's a given with most of today's hosting companies.

2.2. Installation Guide

Setting up the template is a very easy job! Basically, all you have to do is to move (ONLY) the contents of the _template folder to your FTP account, with or without the _documentation folder.

We suggest adjusting the template settings before uploading.

3. Setup guide

Setting up CIRCULAR is a really straight forward job and only involves adjusting the contact form settings and the template's color scheme. After that, your template is ready to use.

3.1. The contact form

The contact form works out of the box and is equipped with a live field validator to make sure all required fields are filled in with proper information. All you need to do is to set up your own e-mail address for delivery and the subject of the e-mail. To do so you need to locate and open for editing the following file: _template/assets/php/contact-form-process.php.

Once you've located and opened the file, look for the following lines of code and adjust them to your needs:

/* Setup your email address to receive emails */
define( 'WEBMASTER_EMAIL', 'username@emailprovider.com' );
define( 'EMAIL_SUBJECT', 'New message received' );

If you wish to edit the default field texts and validation messages, locate and open for editing the following file: _template/assets/js/jquery.validation.settings.js. Throughout this file you will find a series of messages that you can adjust/translate to your needs.

3.2. The color scheme

By default, the template uses a light color setup. Changing it to a dark setup and adjusting the accent colors is basically a job of applying the desired theme classes on the <<body> tag.

For example, if you'd wish to switch to the "dark" environment, you can do so by simply applying the theme-dark class to the body like so:

<body class="theme-dark">

Changing the accent color throughout the template works the same way, by adding the desired theme-class to the BODYtag like so:

<body class="theme-turquoise">

And you can switch to the dark setup and also use a different accent color in the template at the same time by applying both classes to the BODY:

<body class="theme-dark theme-river">

NOTEHere is a list of all color scheme classes available: theme-dark, theme-turquoise, theme-emerald, theme-river, theme-amethyst, theme-carrot, theme-alizarin and theme-asphalt.

Adding your own color scheme is pretty simple. You can simply follow the pattern of any color scheme in the section "6. Color Themes" of the style.css file, copy it, rename the class and add your own color values. Place it at the end of the section and your new color scheme is good to go.

If you've decided for a different than the default accent color, after applying its class to the body you'll need to make a small adjustment to the JavaScript file that handles the display of the circular skill levels. The skill level colors are only defined via JavaScript in the _template/assets/js/custom.circular.js file, where you need to look for and adjust the following function:

function setupPie() {
	$( '.skill-pie' ).easyPieChart( {
		barColor: '#ffbb00',
		trackColor: '#bbbbbb',
		scaleColor: false,
		lineWidth: 5,
		size: 120
	} );
}

4. Usage guide

Below, you will find information on how to set up the information in your template and use the column grid properly so that your resume will look good no matter the resolution or device it is viewed on. Let's start with the foundation: the columns.

4.1. The column grid

The template uses a custom grid. It's a 6-column responsive grid that scales down from 1200px to 320px (mobile devices). It uses padding on columns to create gutters instead of margin and eliminates the need to add classes like last to every last columns in a row. Here's a list of the available column classes and their results:

  • col-full - creates a column that takes up 100% of the available space;
  • col-1-2 - creates a column that takes up 1/2 of the available space;
  • col-1-3 - creates a column that takes up 1/3 of the available space;
  • col-2-3 - creates a column that takes up 2/3 of the available space;
  • col-1-4 - creates a column that takes up 1/4 of the available space;
  • col-3-4 - creates a column that takes up 3/4 of the available space;
  • col-1-5 - creates a column that takes up 1/5 of the available space;
  • col-2-5 - creates a column that takes up 2/5 of the available space;
  • col-3-5 - creates a column that takes up 3/5 of the available space;
  • col-4-5 - creates a column that takes up 4/5 of the available space;
  • col-1-6 - creates a column that takes up 1/6 of the available space;

The each section's content area is surrounded by a <div class="container">, which basically adjusts the template content width based on screen resolution and device and thus, the grid readjusts itself.

Let's look at a few examples of column usage so that you get a grasp of how the grid works

4.1.1. Simple three equal columns row

This can be obtained by using this piece of code

<div class="row">
	<div class="column one-third">First 1/3 column</div>
	<div class="column one-third">Second 1/3 column</div>
	<div class="column one-third">Third 1/3 column</div>
</div>

4.1.2. Three columns, one of 1/2 and two of 1/4

This can be obtained by using this piece of code

<div class="row">
	<div class="col-1-2">1/2 column</div>
	<div class="col-1-4">1/4 column</div>
	<div class="col-1-4">1/4 column</div>
</div>

4.1.3. Nesting columns

In some cases, inside one column you need to add more rows with different numbers of columns inside. The following example creates a two column level-1 layout, a 1/3 column and a 2/3 column. The 2/3 column will hold a couple of rows, the first one with 4 equal 1/4 columns and the second split equally in half. Notice that he nested column will take up the available space, so on level-2 columns you'll get equal fourths of their container which is 2/3 out of the whole template's width.

<div class="row">
	<div class="col-1-3">1/3 column</div>
	<div class="col-2-3">
		<div class="row">
			<div class="col-1-4">1/4 column</div>
			<div class="col-1-4">1/4 column</div>
			<div class="col-1-4">1/4 column</div>
			<div class="col-1-4">1/4 column</div>
		</div>
		<div class="row">
			<div class="col-1-2">1/2 column</div>
			<div class="col-1-2">1/2 column</div>
		</div>
	</div>
</div>

Please note:

We wrap each column set in a <div class="row">. This ensures auto-clearing of all columns inside so that the layout doesn't break.

Now let's have a look at the following elements: header's background image, navigation, skills, services, portfolio items, testimonials and the social media links at the bottom of the template.

4.2.2. Header's background image

The header's background image is added via CSS:

#header {
	position: relative;
	width: 100%;
	height: 70%;
	background-color: #ffffff;
	background-image: url(../img/header.jpg);
	background-repeat: no-repeat;
	background-position: center;
	background-size: cover;
}

Using CSS3 instructions - like background-size: cover; - we ensure the image will stretch proportionally across any screen size. In order to ensure good visibility in older browsers (IE8) with large screen sizes, we suggest using a good quality, large image. If you don't really care about IE8, you can use a smaller image too. For the best effects, using black and white images either very light of very dark are the best choice, like in our demo.

4.2.3. Navigation

Each navigation item consists of two elements: an icon and a text piece. The icons are always hidden in higher resolutions and only show up in smaller resolutions (mobile for example). The texts only show in high resolutions but are replaced by the icons in lower ones:

<li>
	<a href="#section-about">
		<i class="fa fa-user"></i><span>About</span>
		<div class="bullet"><div class="inner-bullet"></div></div>
	</a>
</li>

The icons are generated with the icon font Font Awesome.

4.2.6. Portfolio

The portfolio consists of 2 main sections: the filters area and the projects area. Let's have a look at each of them.

4.2.6.1. The portfolio filters

Here's a sample code extracted from the index.html file which handles the display of filters:

<ul class="filters">
	<li class="filter" data-filter="all">All</li>
	<li class="filter" data-filter="port-website">Website</li>
	<li class="filter" data-filter="port-identity">Identity</li>
	<li class="filter" data-filter="port-mixed">Mixed</li>
</ul>

NOTE The portfolio's filtering capability is handled by a neat jQuery plugin called "MixItUp" about which you can learn more on its dedicated website.

Looking at the code you'll notice the data-filter attribute. Each filter link, once clicked will hide all projects that don't have the class name contained in the data-filter attribute.
